# 📝 FastAPI: Домашнє завдання (Архітектура та RBAC)

Цей проєкт демонструє рефакторинг монолітного застосунку на модульну структуру та впровадження системи прав доступу на основі ролей (Role-Based Access Control).

## 🚀 1. Встановлення

Для запуску проєкту необхідно встановити залежності з файлу `requirements.txt`.

1. **Створіть віртуальне середовище** (рекомендовано):
   ```bash
   python -m venv .venv
   ```
2. **Активуйте його**:
   - Windows: `.venv\Scripts\activate`
   - Mac/Linux: `source .venv/bin/activate`
3. **Встановіть бібліотеки**:
   ```bash
   pip install -r requirements.txt
   ```

---

## 🛠 2. Запуск проєкту

Переконайтеся, що ви знаходитесь у кореневій папці проєкту (там, де файл `main.py`), і виконайте команду:

```bash
uvicorn main:app --reload
```
Після цього API буде доступне за адресою: `http://127.0.0.1:8000`

---

## 🧪 3. Тестування через Swagger UI (/docs)

Це найважливіша частина для перевірки працездатності системи прав доступу.

### Крок 1: Отримання токена (Login)
1. Відкрийте `http://127.0.0.1:8000/docs`.
2. Знайдіть роут **POST `/auth/login`**.
3. Натисніть "Try it out" та введіть одного з користувачів:
   - **Для доступу 403**: `{"username": "simple_user", "password": "123"}`
   - **Для повного доступу**: `{"username": "admin_user", "password": "123"}`
4. Скопіюйте значення `access_token` із результату запиту.

### Крок 2: Авторизація в Swagger
1. Прокрутіть сторінку вгору і натисніть кнопку **"Authorize"** (з іконкою замка).
2. У поле **Value** вставте скопійований токен.
3. Натисніть **Authorize**, а потім **Close**.

### Крок 3: Перевірка ролей на коментарях
1. **Створення/Видалення**: Спробуйте виконати `POST` або `DELETE` у секції коментарів.
   - Якщо ви залогінені як `simple_user`, сервер поверне **403 Forbidden**.
   - Якщо як `admin_user` або `author_user`, запит пройде успішно (код 200).
2. **Читання**: Роут `GET /news/{news_id}/comments` працює без токена для всіх.

---

## 📂 Структура файлів
- `main.py` — ініціалізація та підключення роутерів.
- `db.py` — імітаційна БД (словники користувачів, новин та коментарів).
- `schemas.py` — Pydantic-моделі (UserLogin, CommentCreate тощо).
- `security.py` — перевірка JWT-токена та функція `get_author_or_admin`.
- `routers/` — логічне розділення API на модулі:
  - `auth.py` — логін.
  - `news.py` — новини.
  - `comments.py` — коментарі (захищені ролями).

---

## ⚠️ Важливо
При спробі звичайного юзера (`simple_user`) видалити або змінити коментар, система **обов'язково** повертає статус-код **403**, як того вимагають умови завдання.
